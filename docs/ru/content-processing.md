# Проект content-neuro-update 

**Задача:** Массовая обработка контента 4000 статей на Wordpress по 20+ правилам.

**Сложность:** Объёмные статьи с техническими инструкциями, в которых необходимо сохранить точность данных.

**Проблема:** При обработке с помощью нейросетей терялись инструкции, правила постоянно вылетали из контекста.

**Решение:** Система автоматизированной обработки WordPress-постов с использованием декларативных правил и вспомогательных скриптов обрабатывающих контент по этим правилам.

![Воркфлоу разработки правил](/ru/images/content-processing/rule-creation-workflow.webp)

:::tip
> **Воркфлоу обработки контента:**
> 1. Выборка постов для обработки
> 2. Скачивание постов на локальную машину
> 3. Фиксация содержимого в Git
> 4. Обработка постов с помощью утилит по необходимым правилам
> 5. Проверка изменений через Git diff
> 6. Откат изменений в случае необходимости или ручная корректировка
> 7. Аплоад статей с изменениями

:::

**Результат:** Сохранение десятка человеко-часов на обработку контента.

**Эволюция проекта**

После старта проекта, Google Antigravity интегрировал в свой IDE технологию создания скиллов, что по сути является аналогом моей идеи детерминирования правил обработки контента. Приемущества SKILL.md является в том, что нейросеть нативно работает по заданным правилам, при необходимости "в ручную" исправляя проблемы пропущенные вспомогательными утилитами.

На данный момент воркфлоу переводится на использование [Agent Skills](https://agentskills.io/specification).

### Сложность контента

Статьи представляют собой **объёмные технические инструкции** с характерными особенностями:

- Большой объём текста (до 50+ экранов)
- Множество блоков кода и терминальных команд
- Скриншоты с пошаговыми действиями
- Сложные вложенные списки
- Таблицы с техническими параметрами
- Перекрёстные ссылки между разделами

### Проблема: Почему стандартные подходы не работают

При попытке обработки с помощью нейросетей напрямую возникали критические проблемы:

| Проблема | Описание |
|----------|----------|
| **Потеря контента** | • Части инструкций исчезали при обработке<br>• Блоки кода обрезались или искажались<br>• Изображения теряли атрибуты |
| **Потеря контекста правил** | • Правила "вылетали" из памяти модели<br>• На 2-3 статье модель начинала игнорировать правила<br>• Приходилось постоянно напоминать инструкции |
| **Непредсказуемость** | • Одно и то же правило применялось по-разному<br>• Невозможно гарантировать консистентность<br>• Сложно отследить что именно изменилось |
| **Отсутствие версионирования** | • Нет возможности отката изменений<br>• Нельзя применить правила инкрементально<br>• Потеря данных при ошибках |

### Ключевой инсайт

:::info
> **Нельзя отправить 4000 статей в нейросеть и ожидать корректного результата.**
> 
> Необходима **система**, которая:
> 1. Изолирует каждую статью для обработки
> 2. Гарантирует применение правил без потери контекста
> 3. Сохраняет оригинальный контент без искажений
> 4. Версионирует все изменения
:::

## Проблема

Массовая обработка контента WordPress-сайтов требует значительных временных и человеческих ресурсов:

- **Однотипные правки** — исправление пунктуации, форматирования, выравнивания изображений
- **Человеческий фактор** — ошибки при ручной обработке сотен постов
- **Отсутствие стандартизации** — разные авторы оформляют контент по-разному
- **Трудозатраты** — часы работы редактора на типовые задачи

## Решение: Content Neuro Update

**Content Neuro Update** — это система автоматизированной обработки контента WordPress-постов с использованием AI-агента и декларативных правил.

### Ключевая инновация

:::tip
Вместо написания кода для каждой задачи, мы создаём **декларативные правила на естественном языке** (Markdown), которые AI-агент интерпретирует, создаёт вспомогательные утилиты, и применяет к контенту.

```
📝 Правило (Markdown) → 🤖 AI-агент → 📄 Обработанный контент
```
:::

```python
# Сложный код для каждой задачи
def fix_punctuation(html):
    soup = BeautifulSoup(html, 'html.parser')
    for li in soup.find_all('li'):
        # ... 50+ строк логики
```

### Наш подход (правила):

```markdown
# Правило 9: Правильная пунктуация списков

## Основные правила:
1. Все элементы списка, кроме последнего → `;`
2. Последний элемент → `.`

## Примеры:
### До:
- Первый элемент,
- Второй элемент.

### После:
- Первый элемент;
- Второй элемент.
```

## Рабочий процесс (Workflow)

### Процесс создания правила

1. **Текстовое правило** — Первичное описание на естественном языке (`Rules/N_rule.md`)
2. **Улучшение через AI** — Нейросеть дополняет правило крайними случаями (исключения, особые ситуации, нюансы)
3. **Создание тестов** — Тесткейсы для валидации правила (`tests/test_N_rule.py`)
4. **Утилита обработки** — Рабочий скрипт на основе тестов (`utils/N_rule/*.py`)

## Ключевые особенности

### 1. Двунаправленная синхронизация

```bash
# Скачать посты с сайта
python main.py download

# Загрузить изменения обратно
python main.py upload --mod
```

### 2. Интеллектуальная загрузка

- `upload` — загрузка всех постов
- `upload --mod` — только git-modified файлы (инкрементальная загрузка)

### 3. Безопасность

- Автоматическое создание ревизий WordPress
- Git-контроль всех изменений
- Возможность отката на любую версию

### 4. Гибкость правил

Каждое правило может содержать:
- Описание задачи
- Пошаговые инструкции
- Примеры "до/после"
- Исключения и особые случаи
- Тесткейсы для проверки

### 5. Вспомогательные инструменты

- Анализаторы контента (`utils/utils/`)
- Тесты для валидации правил (`tests/`)
- Скрипты автоматизации (`utils/*_rule/`)

## Масштабирование
